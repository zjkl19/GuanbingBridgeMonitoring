# 桥梁健康监测工作台使用说明

## 日常使用

1. 打开 `桥梁健康监测工作台.exe`。
2. 选择桥梁项目和本机路径配置组。程序默认按电脑名自动匹配；也可选择“自定义”并填写数据目录。内置存储位置只包含开发机、133 分析机和 126 数据机；办公室电脑不作为数据存储或计算方案。
3. 选择监测日期和需要处理的模块。
4. 点击“检查配置与路径（不运行）”，确认数据、配置和执行程序可用。
5. 点击“启动本机分析”。普通用户保持“自动识别”即可：有 CSV 时读取 CSV；没有 CSV 时读取有效 MAT 缓存。
6. 分析完成后，先在主页面“本次计算结果在哪里”查看实际保存位置，再到“结果与图件审核”页检查模块状态、正式图件数据完整性和源数据缺口说明。
7. 确认图件后进入“报告生成”页。报告在同一工作台内后台生成，不会再弹出独立报告程序。

## 任务按钮说明

- “检查配置与路径（不运行）”：只做路径、配置和执行环境检查，不开始计算。
- “保存任务方案（便于恢复）”：保存当前桥梁、目录、日期、模块和报告设置，不开始计算。
- “打开已保存任务方案”：恢复之前保存的设置和任务状态。
- “查看任务历史”：查看本机该数据目录下的历史任务及是否可恢复。
- “请求停止”：向当前后台分析写入安全停止请求；正在保存的结果可能需要短时间才能结束。

## 数据读取说明

- “自动识别（推荐）”：有 CSV 时使用 CSV；CSV 不存在时使用经校验的 MAT 缓存。
- “仅读取 MAT”属于高级隔离验证方式，普通用户无需设置。
- 程序不会因图形连续显示而伪造缺失的原始日期或样本；实际缺口会在结果检查和报告中披露。

## 分析结果保存在哪里

- 主页面“本次计算结果在哪里”显示当前任务的实际结果根目录。普通本机任务通常就是所选数据根目录；隔离验证或候选任务则写入任务指定的独立候选目录，不会回写原数据目录。
- 结果根目录下的 `stats` 保存统计工作簿，`run_logs` 保存运行状态、任务记录和数据来源记录；正式图件按分析模块保存在结果根目录下对应的图件目录中。
- 可直接点击“复制路径”“打开结果目录”“打开统计表”或“打开运行记录”，无需再到文件资源管理器中猜测路径。任务完成后，进度区和日志也会再次写明实际结果目录。任务历史页可打开该任务的计算结果目录。
- 任务运行后即使修改了桥梁、数据目录或日期来准备下一次任务，结果区仍固定显示当前任务的真实目录；帮助文字会另行显示下一次任务的计划目录，避免把两个任务混淆。
- 报告 DOCX/PDF 是单独的报告输出，不等同于分析结果根目录；其实际位置以“报告生成”页显示的输出目录和生成记录为准。

## 预生成分析缓存

- “预生成分析缓存”面向所有已配置桥梁，只处理当前桥梁分析配置实际会用到的 CSV；它不是递归转换目录中所有 CSV 的工具。
- 工作台根据数据布局自动选择三种处理方式：九龙江/水仙花日导出布局 `jlj_daily_export`、普通日期目录 `dated_folders`、洪塘周期目录 `hongtang_period`。
- 该功能默认不勾选；只在需要加速后续反复分析且磁盘空间充足时由用户主动开启。
- 缓存保留原始样本，不清洗、不滤波、不降采样。普通缓存预生成默认保留全部源 CSV；WIM、洪塘低频 Excel、未配置 CSV 和其它非 CSV 数据不在预生成范围内。

### 缓存完成后释放 CSV 空间（高风险、默认关闭）

1. 该选项支持三种具备原 ZIP 恢复证据的数据布局：九龙江/水仙花日导出 `jlj_daily_export`、普通日期目录 `dated_folders` 和洪塘周期目录 `hongtang_period`。界面会按当前桥梁的数据布局及实际数据根做只读预检。
2. 必须把它作为独立预处理任务运行。允许同时选择“压缩包预检查”“批量解压”“预生成分析缓存”，不得混入分析模块、批量重命名 CSV、去表头或重采样。完成后再新建分析任务。
3. 勾选选项后，还必须完整输入确认口令 `DELETE_VERIFIED_EXTRACTED_CSV`；仅勾选复选框不会删除文件。该授权只保存在当前任务方案中，不写入桥梁公共配置。
4. 同时选择解压和缓存时，程序按自然日依次执行：安全解压该日 ZIP、生成或复用该日缓存、校验 MAT/元数据缓存对、核验原 ZIP 恢复来源、写入持久收据、删除该日适用 CSV。不会先解压整月后再统一删除。
5. 只选择缓存时，只能处理已经存在且仍有有效逐日解压清单和原 ZIP 的目录；缺少原 ZIP、解压清单或恢复证据时会拒绝删除。
6. 只有配置实际使用、缓存可独立读取并且能由原 ZIP 恢复的 CSV 才会删除。普通日期目录/洪塘周期目录同日同类允许多个 ZIP，但每个待删 CSV 必须按相对路径、字节数和 CRC 唯一匹配一个 ZIP 条目；任何配置源缺失、匹配歧义、ZIP/缓存变化或缓存不可读都会使该日整日零删除。原 ZIP、WIM、Excel、未配置 CSV、非 CSV、MAT 缓存和收据始终保留。
7. `jlj_daily_export` 每日收据保存在对应日分区的 `.bms_cache_source_cleanup_receipt.json`；普通日期目录/洪塘周期目录的每日收据保存在数据根 `run_logs/cache_source_cleanup_receipts/standard_YYYYMMDD.json`。缓存 worker 不删除文件；只有父进程在整日核验闭合后提交。某日失败会停止整批任务，后续分析不会继续。
8. 删除阶段若意外中断，不要手工编辑或删除收据，也不要直接清理临时文件。保持原 ZIP 和缓存不变，使用同一数据根、配置和任务选项续跑；程序会依据收据核对已完成和待恢复状态。非 `committed` 状态必须先查明原因。
9. 该流程减少已解压 CSV 占用，但仍保留 ZIP 和 MAT，因此不能替代启动前容量检查及运行期间的磁盘余量监控。
10. 安全清理完成后，普通用户的新分析任务继续保持“自动识别（推荐）”即可；CSV 不存在时程序会读取有效 MAT。只有正式隔离验收才需要选择“仅读取 MAT”。

## 从曲线设置清洗阈值

1. 进入“配置与预警值 → 数据清洗阈值”，先在表中选择一条“测点专用”规则。
2. 点击拖线或框选入口后，工作台会按当前桥梁、数据根目录、日期、配置、模块和测点自动匹配当前任务曲线预览。普通流程无需选择任何文件；只需核对窗口顶部显示的桥梁、测点和月份是否正确。若没有找到完全匹配的预览，可点击“没有曲线？关闭并前往‘自动清洗建议’生成”，工作台会切换到建议页并只勾选当前分析类型；生成后返回重试。
3. 需要参考其它任务、其它项目或历史成果时，优先点击“直接选择 MATLAB FIG（旧方式）”，可选择任意可信 `.fig`，再在 MATLAB Runtime 原生窗口中选择坐标轴/曲线并拖线或框选。工作台会核对 FIG 路径、大小和 SHA-256，交互期间文件变化则拒绝采用结果。JPG/PNG 不含可复核 X/Y 数据，不能用于该入口。
4. “高级：导入系统曲线记录 JSON”保留给工作台生成的曲线记录，并在包含多条曲线时由用户选择一条。任意 FIG 或外部 JSON 都只作为当前选中配置行的数值参考：界面醒目标注“未绑定当前任务”，不会冒充当前任务校验通过。拖线默认只采用上下限并保留当前配置行原有时间窗；只有确认 FIG 与当前任务一致时，才勾选“同时采用 FIG 中拖线得到的时间窗”。框选使用 FIG 中真实有限样本，且不据此外推当前任务删除量。
   直接 FIG 操作期间会显示独立进度窗口；如选错文件或 MATLAB Runtime 窗口无法继续，可点击“停止本次 FIG 操作”。停止只终止本次由工作台启动的 FIG 交互进程，不影响正在运行的正式分析任务，也不会修改 FIG 源文件。
5. 需要同时设置上下限时，点击“拖线设置上下限（旧 MATLAB 方式）”。该入口会打开曲线预览并拖线设置上下限：黄色下限线以下、红色上限线以上的数据会被清洗；当前任务曲线的两条线始终保存同一个时间窗，默认取预览首尾时间，拖动紫色边界可同步调整，也可在数值框中精确输入。
6. 只需从异常点取一侧边界时，使用两个独立框选按钮：
   - “框选设为下限”：下侧框选取框中实际有限样本的最高值，删除严格低于该值的数据；
   - “框选设为上限”：上侧框选取框中实际有限样本的最低值，删除严格高于该值的数据。
7. 等于候选阈值的点保留。候选值来自框内真实样本，不来自矩形边缘；框选横向范围只用于选择候选样本，不会暗中改变当前规则时间窗。
8. “滤波后二次清洗”页也可直接选择任意可信 FIG；为防止口径混淆，该页不自动加载滤波前的任务预览，也不开放滤波前系统 JSON。请确保所选 FIG 确实对应目标滤波结果。
9. 确认前会显示框中样本数和候选阈值。当前任务抽样预览可以估算删除数；外部 FIG 不据此外推当前任务删除量。取消或关闭窗口不修改表格；确认后也只形成尚未保存的表格修改，可用“撤销上次尚未保存的曲线设置”恢复。
10. 真正写入配置仍需点击页面底部“覆盖保存（自动备份）”或“保存副本”。预览可能经过抽样，正式删除数量和比例必须以完整缓存重算为准。

## 报告与更新

- 默认启动时自动检查 GitHub 更新，可在界面中关闭。
- 报告生成、逐页渲染和质量检查都在统一工作台内完成。
- 若修改了数据目录、日期、模块或配置，旧的审核状态会自动失效，必须重新加载对应分析结果，防止把不同任务的数据混入报告。
