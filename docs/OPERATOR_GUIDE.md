# 桥梁健康监测工作台使用说明

## 日常使用

1. 打开 `桥梁健康监测工作台.exe`。
2. 选择桥梁项目和本机路径配置组。程序默认按电脑名自动匹配；也可选择“自定义”并填写数据目录。内置存储位置只包含开发机、133 分析机和 126 数据机；办公室电脑不作为数据存储或计算方案。
3. 选择监测日期和需要处理的模块。
4. 点击“检查配置与路径（不运行）”，确认数据、配置和执行程序可用。
5. 点击“启动本机分析”。普通用户保持“自动识别”即可：有 CSV 时读取 CSV；没有 CSV 时读取有效 MAT 缓存。
6. 分析完成后，在“结果与图件审核”页检查模块状态、正式图件数据完整性和源数据缺口说明。
7. 确认图件后进入“报告生成”页。报告在同一工作台内后台生成，不会再弹出独立报告程序。

## 任务按钮说明

- “检查配置与路径（不运行）”：只做路径、配置和执行环境检查，不开始计算。
- “保存任务方案（便于恢复）”：保存当前桥梁、目录、日期、模块和报告设置，不开始计算。
- “打开已保存任务方案”：恢复之前保存的设置和任务状态。
- “查看任务历史”：查看本机该数据目录下的历史任务及是否可恢复。
- “请求停止”：向当前后台分析写入安全停止请求；正在保存的结果可能需要短时间才能结束。

## 数据读取说明

- “自动识别（推荐）”：有 CSV 时使用 CSV；CSV 不存在时使用经校验的 MAT 缓存。
- “仅读取 MAT”属于高级隔离验证方式，普通用户无需设置。
- 程序不会因图形连续显示而伪造缺失的原始日期或样本；实际缺口会在结果检查和报告中披露。

## 预生成分析缓存

- “预生成分析缓存”面向所有已配置桥梁，只处理当前桥梁分析配置实际会用到的 CSV；它不是递归转换目录中所有 CSV 的工具。
- 工作台根据数据布局自动选择三种处理方式：九龙江/水仙花日导出布局 `jlj_daily_export`、普通日期目录 `dated_folders`、洪塘周期目录 `hongtang_period`。
- 该功能默认不勾选；只在需要加速后续反复分析且磁盘空间充足时由用户主动开启。
- 缓存保留原始样本，不清洗、不滤波、不降采样，不移动或删除源 CSV/ZIP。WIM、洪塘低频 Excel、未配置 CSV 和其它非 CSV 数据不在预生成范围内。

## 从曲线设置清洗阈值

1. 进入“配置与预警值 → 数据清洗阈值”，先在表中选择一条“测点专用”规则。
2. 需要同时设置上下限时，点击“拖线设置上下限（旧 MATLAB 方式）”。必须先加载并人工核对当前桥梁、测点和月份的曲线预览；该入口会打开曲线预览并拖线设置上下限：黄色下限线以下、红色上限线以上的数据会被清洗；两条线始终保存同一个时间窗，默认取预览首尾时间，拖动紫色边界可同步调整，也可在数值框中精确输入。
3. 只需从异常点取一侧边界时，使用两个独立框选按钮：
   - “框选设为下限”：下侧框选取框中实际有限样本的最高值，删除严格低于该值的数据；
   - “框选设为上限”：上侧框选取框中实际有限样本的最低值，删除严格高于该值的数据。
4. 等于候选阈值的点保留。候选值来自框内真实样本，不来自矩形边缘；框选横向范围只用于选择候选样本，不会暗中改变当前规则时间窗。
5. 确认前会显示框中样本数、候选阈值、预计删除数和抽样提示。取消或关闭窗口不修改表格；确认后也只形成尚未保存的表格修改，可用“撤销上次尚未保存的曲线设置”恢复。
6. 真正写入配置仍需点击页面底部“覆盖保存（自动备份）”或“保存副本”。预览可能经过抽样，正式删除数量和比例必须以完整缓存重算为准。

## 报告与更新

- 默认启动时自动检查 GitHub 更新，可在界面中关闭。
- 报告生成、逐页渲染和质量检查都在统一工作台内完成。
- 若修改了数据目录、日期、模块或配置，旧的审核状态会自动失效，必须重新加载对应分析结果，防止把不同任务的数据混入报告。
