# v1.8.0

`v1.8.0` is the first stable release of the unified PySide6 bridge-health
monitoring workbench. It keeps the validated MATLAB analysis engine as an
independent backend process while unifying operator workflows, local task
management, result review and report generation in one Chinese Windows EXE.

## Highlights

- One operator entry point: `桥梁健康监测工作台.exe`. Report generation runs as
  an embedded headless worker; the package no longer contains or opens a
  standalone report-builder EXE.
- The project catalog is data-driven and supports all configured and future
  bridges without a fixed bridge count. Machine path profiles select known
  data roots by computer name and retain a custom profile for new machines.
- Normal analysis uses `auto` source selection: valid CSV is authoritative when
  present, otherwise a compatible MAT cache is used. Strict `mat_only` remains
  available for isolated verification and never falls back to CSV.
- PySide6 now covers warning-value review, cleaning and post-filter cleaning,
  automatic-cleaning proposals, zero correction, grouped plots, shared plot
  parameters and spectrum overrides with guarded JSON writes and backups.
- Local tasks are recoverable from bounded task history. User-facing labels use
  plain Chinese, and the packaged application includes a native icon, operator
  guide, automatic stable-release checks and transactional update rollback.
- Release packaging uses a schema-v3 closed inventory and blocks publication
  unless compiled analysis, embedded reporting, all-profile configuration,
  CLI, native GUI screenshot, report-condition and visual-QA smokes pass.

## Analysis and report corrections

- Hongtang Q2 uses the completed June 28-30 supplement, corrected natural-day
  earthquake coverage, calibrated acceleration-spectrum ranges, completed wind
  statistics and real Word page-number fields. Actual source gaps and the CS8
  cable-acceleration RMS threshold exceedance remain explicitly disclosed.
- Guanbing report tables, units and OOXML media handling are hardened and tied
  to the selected task/result context.
- Shuixianhua monthly reports reject stale template-period text or figures,
  lock current-period media, resolve wind summaries uniquely by point and
  requested start/end dates, and fail closed when report QC cannot complete.
  Cable-force conversion remains unavailable without an approved per-cable
  effective-length and linear-density table.
- Jiulongjiang monthly reports now validate the requested monitoring period,
  reject figures and wind summaries from another month, bind matching patrol
  material only, clear stale patrol content when no matching source exists,
  remove unused template media and fail closed on QC errors.

## Operational notes

- Source-data gaps are never interpolated or silently presented as complete.
- Microsoft Word output is authoritative for reports that use combined
  `STYLEREF`/`SEQ` fields. Formal PDFs are generated by the report builder or
  the unified isolated Word exporter; LibreOffice output is a layout preview
  only and is never presented as the authoritative PDF.
- Production upgrades must stage and verify the complete ZIP before switching
  the live directory. Keep the previous production tree and task/shortcut
  snapshots for at least 7-14 days or one to two completed production cycles.
- Existing project configuration files are preserved by the transactional
  updater. Review configuration-schema changes explicitly rather than relying
  on an overlay replacement.
