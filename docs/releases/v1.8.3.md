# Bridge Health Monitoring Platform v1.8.3

Release type: stable patch release.

## Project identity

The repository is renamed from `GuanbingBridgeMonitoring` to
`BridgeHealthMonitoringPlatform`. The former name reflected the first bridge
supported by the codebase; the platform now supports multiple independent
bridges and report types. The user-visible Windows product name remains
`桥梁健康监测工作平台`.

Compatibility is deliberate:

- MATLAB/Python package names and settings identities are unchanged;
- the existing `BridgeMonitoringWorkbench-*` update asset prefix is retained;
- current and legacy executable names remain accepted by the updater;
- GitHub redirects the former repository URL to the new repository;
- installed application directories do not need to be renamed during upgrade.

The verified extracted-source cleanup contract is unchanged: it remains
default-off, requires the exact `DELETE_VERIFIED_EXTRACTED_CSV` confirmation,
and deletes only the configuration-bound allowlist after cache and archive
proof. ZIP sources, WIM inputs and unconfigured CSVs remain outside that
deletion boundary.

## Fixes

- Jiulongjiang monthly acquisition reporting accepts a retained MAT cache only
  when its transactional `jlj_csv_v2` metadata closes schema, configuration,
  pair identity, exact MAT byte count and original non-empty source evidence.
  A currently present but record-free CSV is authoritative and is not hidden by
  an older retained cache.
- Reviewed Jiulongjiang source-coverage notes are bound into the report build
  manifest and the analysis-manifest-SHA disclosure workflow.
- Duplicate disclosure labels are removed from generated report prose.
- Large Word reports update fields and save DOCX in one Word session, then
  export PDF through a fresh PowerShell/COM process using Word `SaveAs2` format
  17. This prevents the Office 16 field-update/export hang while preserving
  authoritative Word output.
- Frozen invalid-CLI calls tolerate missing, closed or invalid inherited
  standard-error handles and fall back to the diagnostic log instead of opening
  an English PyInstaller exception dialog.

## Validation requirements

Promotion requires the complete Python and MATLAB suites, all bridge-profile
configuration checks, packaged CLI and embedded-report smokes, native Windows
focus/DPI/icon/screenshots, and real-data report acceptance for Guanbing and
Hongtang. Reports with real source gaps must retain
`passed_with_disclosures`; this release does not convert data gaps into clean
passes or change engineering/statistical conclusions.

The public-release diff guard must close the exact v1.8.3 allowlist with no
forbidden internal operations files or sensitive added lines before publication.
