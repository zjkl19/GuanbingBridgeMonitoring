# v1.8.0-rc1

`v1.8.0-rc1` is the first public release candidate for the PySide6 bridge
health-monitoring workbench. It is intended for isolated parallel validation;
the legacy MATLAB GUI remains the production fallback during the observation
period.

## Highlights

- Adds the packaged Chinese `桥梁健康监测工作台.exe` while retaining the
  validated MATLAB numerical-analysis kernel and Python report builders.
- Unifies project/date/module selection, local analysis launch and recovery,
  warning/cleaning/filter/offset/group/plot/spectrum configuration, result and
  figure review, report generation and page-level quality inspection.
- Replaces the former fixed “六桥自检” contract with catalog-driven
  “所有桥梁自检”. Every profile in `config/bridge_profiles.json` must run in the
  frozen EXE and match its configuration, template, modules and report
  capability; adding a bridge automatically expands the release gate.
- Adds transactional GitHub updates, whole-package SHA256 inventory checking,
  explicit backup retention and exact fault-injected rollback.
- Corrects SemVer prerelease precedence so `v1.8.0-rc1` can later receive the
  stable `v1.8.0` update.
- Includes the audited Hongtang acceleration-spectrum search intervals and Q2
  report-template/page-field corrections from the development line.

## Validation

- Python regression: `297/297` passed.
- MATLAB catalog contract and legacy-GUI smoke: passed.
- Frozen EXE: every configured bridge profile passed; catalog/config/template
  assets remained byte-identical.
- Native GUI: `v1.8.0-rc1` displayed “所有桥梁自检”, and the installed matrix
  closed at the current catalog count with all checks passed.
- Real ZIP update cycle: 599-file inventory, SHA256, Chinese EXE replacement,
  configuration/unmanaged-file retention, stale-runtime removal, native
  screenshot, backup creation and exact rollback all passed.

## Deployment boundary

- Install the candidate into an isolated directory such as
  `F:\Guanbing_v1.8.0-rc1`.
- Do not overwrite or auto-update the existing `F:\Guanbing` production tree.
- Use separate scheduled-task names, status/log roots and candidate outputs.
- Hongtang Q2 analysis/report generation and Guanbing analysis are the first
  production-data comparison cases on 133.
