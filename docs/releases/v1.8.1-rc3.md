# v1.8.1-rc3

`v1.8.1-rc3` is the local reviewed successor to rc2. It keeps the
user-facing cache-prebuild module and the operator controls introduced in rc2,
while closing data-integrity and lifecycle defects found during release review.
It remains a development candidate and is not an authorization to modify the
isolated Jiulongjiang acceptance workflow on machine 133.

## Candidate changes

- Rejects a source ZIP that changes after capacity planning, binds the exact
  path/size/CRC entry set to the archive opened for extraction, and verifies
  the archive snapshot again before atomic publication.
- Replaces age-only extraction locks with a host/PID/UUID lease. A live local
  task cannot be reclaimed by age, stale recovery remains explicit, owner
  write failures clean up safely, and an old cleanup token cannot delete a new
  task's lock.
- Preserves terminal analysis/report results across status refresh and
  stop-versus-natural-completion races.
- Requires SHA-256 as well as byte count for strict report artifact binding.
- Prevents an RMS-only refresh from overwriting raw grouped time-history plots
  with synthetic zero-valued lines.
- Adds a non-disruptive Qt offscreen audit-build path for use while the desktop
  is occupied. Offscreen screenshots are explicitly marked non-native and
  cannot satisfy the formal release-package gate.
- Recomputes every frozen distribution file's path, byte count and SHA-256
  immediately before compression, and requires all embedded version records to
  match the requested release version.
- Removes the low-capability office PC from the shared storage-location
  profiles while retaining it in the operations inventory for light remote
  maintenance only.
- Makes raw CSV cache prebuild an optional, unchecked-by-default capability for
  every configured bridge. `jlj_daily_export`, `dated_folders` and
  `hongtang_period` are dispatched to their matching cache contracts; only
  configured analysis CSVs are eligible and source CSV/ZIP files are never
  moved or deleted.
- Makes the cleaning-page entry explicit: operators open a curve preview and
  drag the lower-bound line to delete values below it, or the upper-bound line
  to delete values above it. Exact entry, cancel and undo remain available.
