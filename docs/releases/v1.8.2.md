# v1.8.2

`v1.8.2` is the stable follow-up to `v1.8.1`. It keeps the validated MATLAB
analysis kernel and statistical definitions, while completing the direct FIG
threshold workflow, verified cache lifecycle, large-data recovery and Word
report release gates exercised during the development cycle.

## Operator-visible changes

- The unified workbench can use an arbitrary trusted MATLAB `.fig` as a
  numerical reference for point-specific cleaning thresholds. Double-line
  dragging keeps values inside the lower/upper bounds. The lower-side box takes
  the highest finite sample in the selected rectangle; the upper-side box takes
  the lowest finite sample. Values equal to a threshold are retained.
- Current-task previews and advanced JSON import remain available. External
  FIG selection is deliberately separate and records the source path, size,
  modification time and SHA-256 before and after the interaction.
- Cache prebuild is available for all configured bridge profiles. Verified
  deletion of extracted CSV remains default-off, requires
  `DELETE_VERIFIED_EXTRACTED_CSV`, and only deletes the exact configured CSVs
  covered by independently loadable MAT/meta pairs and unchanged recovery ZIP
  evidence. ZIP, WIM, Excel, unconfigured CSVs, cache and receipts are retained.

## Large-data and recovery safety

- Full-source statistics remain unchanged. Formal high-frequency plots use the
  accepted bounded peak-preserving rendering path; batch figures are hidden
  during generation and saved `.fig` files reopen visibly for an operator.
- High-memory recovery isolates cable-acceleration points and spectrum modules
  in fresh Runner processes. A memory failure closes only figures created by
  the failing step and prevents predictable chained high-memory failures.
- Analysis manifests are written compactly through a temporary file, decoded
  again before atomic publication, and fall back to a small valid failed
  manifest rather than a truncated success-looking file.

## Report integrity

- Monthly report jobs can carry an audited source-quality note from the saved
  task context into the Zhishan report. Missing natural-day partitions and
  missing adjacent rolling-export files are therefore disclosed in the final
  report instead of living only in an analysis log.
- Bearing-displacement manifests now register the producer's four concrete
  raw/filtered single/group output directories. The statistics workbook and
  every formal JPG/FIG are hashed into the analysis manifest rather than only
  the workbook being listed.
- The Guanbing monthly report removes stale calendar text and unused historical
  media from sample-period output. Report numbers are normalized across body,
  headers, footers and text boxes; a missing or inconsistent number fails
  generation or quality control instead of silently passing.
- Single-section body page fields use deterministic `PAGE` plus
  `SECTIONPAGES`. Multi-section bodies use `PAGE` plus a `PAGEREF` to a hidden
  bookmark on the final body page, so portrait/landscape section changes do
  not reset the displayed total. Chinese field text uses `楷体_GB2312`, Latin
  digits use Times New Roman and the size is small-five (`9 pt`). Microsoft
  Word refreshes body fields before a final header/footer refresh and
  repagination so later updates cannot invalidate the displayed totals.
- Zhishan PSD report media are selected from the requested report month. Day
  10 remains the preferred representative day for continuity; if it is
  missing, fallback is restricted to the same month and never reuses stale
  March media.
- Ten-minute wind-speed figures distinguish their derived interval samples
  from the raw source sample count in formal provenance. The plotted curve and
  statistics are unchanged, while strict source/input/finite/plotted checks
  now close against the actual 10-minute series.
- The packaged Guanbing template is protected by an explicit binary contract:
  the DOCX member inventory, order, sizes, hashes and the single authorized
  footer XML change are verified. Uncontracted binary changes fail closed.

## Required promotion checks

1. Run the complete Python and MATLAB suites, report-template checks and the
   public-release diff guard from the exact committed source. The health check
   runs the full non-GUI core tests, verified-CSV cleanup contracts and native
   GUI smoke in separate MATLAB processes with separate preference roots.
2. Build and package from a clean tree without skipping the compiled Runner or
   native Windows screenshots. Verify source, distribution manifest and ZIP
   hashes for the template and Runner.
3. Run real cached-data end-to-end regressions serially for the selected bridge
   periods. Validate statistics, formal-plot source records, month axes and
   required missing-data disclosures.
4. Generate reports through the unified workbench, update fields and export PDF
   with Microsoft Word, render every page, and reject stale period text, broken
   fields, blank pages, clipping or inconsistent report numbers.
5. Deploy the exact accepted package to an isolated directory, keep a rollback
   copy, rehash every file and repeat CLI/GUI/report smoke checks before a
   production cutover.
