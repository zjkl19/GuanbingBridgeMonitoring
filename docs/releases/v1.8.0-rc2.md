# v1.8.0-rc2

`v1.8.0-rc2` is the second isolated release candidate for the PySide6 bridge
health-monitoring workbench. The validated MATLAB analysis kernel remains in
place; the former MATLAB GUI remains available as a temporary fallback while
production-data comparisons continue.

## Highlights

- Adds hostname-matched storage-location groups with manual and per-task custom
  overrides, while keeping one canonical business configuration per bridge.
- Adds direct, source-aware editing of warning bounds, warning levels and plot
  reference lines; empty two-sided bounds are explained instead of shown as a
  blank or failed page.
- Replaces inverted-threshold data suppression with explicit dated exclusion
  ranges carrying a readable reason.
- Migrates active spectrum configurations to `peak_orders`; Hongtang
  acceleration searches now start at least `0.05 Hz` above the corresponding
  theoretical frequency.
- Applies the confirmed Hongtang acceleration and cable-acceleration cleaning
  limits and adaptive cable-acceleration axes.
- Adds a dedicated application icon, Windows taskbar identity, the organization
  logo, clearer task controls and larger UI text.
- Adds Hongtang wind-table `10min` mean-wind maximum statistics and a real Word
  body-page formula while retaining the existing wind-rose sampling method.
  The formula uses the template's three actual unnumbered front-matter pages;
  authoritative Word rendering verifies `第 1 页 共 78 页` through
  `第 78 页 共 78 页`.
- Makes report input selection fail closed: the exact analysis manifest,
  optional derived-input sidecar, result root, artifact sizes/hashes and data
  coverage audit are pinned. Missing or warning-bearing strict reports can no
  longer be presented as successfully checked.
- Compares acceleration RMS statistics with their configured report threshold
  instead of emitting a fixed conclusion. The supplemented Hongtang Q2 result
  therefore reports CS8 `2.468 m/s2` as above the stated `1.000 m/s2`
  first-level threshold and requests original-time-history/alarm review.

## Validation boundary

- Build and regression evidence is recorded in `docs/current_task_state.md`.
- Hongtang Q2 and Guanbing June validation use isolated local `mat_only` views;
  they do not run unzip, CSV preprocessing, resampling, low-frequency sync or
  WIM and do not overwrite the downloaded input trees.
- The Hongtang Q2 June 28-30 supplement completed all nine selected modules,
  47/47 formal plot source/count checks, 340/340 Python tests and 99/99 focused
  MATLAB tests. The final package contains 605 inventory-verified files and
  passes compiled analysis/report, CLI, GUI and native screenshot smoke gates.
  The final Word-authoritative report renders 81 physical pages, with only the
  intentional blank cover verso, no clipped page, and no broken reference.
  Its end-to-end manifest records the actual three front-matter pages,
  `status=ok`, zero missing/warning items and all 186 bound report images.
- Do not merge into `main` or overwrite the existing production installation
  until the candidate comparison and rendered-report checks are complete.
