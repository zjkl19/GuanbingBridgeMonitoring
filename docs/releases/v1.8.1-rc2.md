# v1.8.1-rc2

`v1.8.1-rc2` is a local-only development candidate built after the isolated
`v1.8.1-rc1` Jiulongjiang production trial had already started. It must not be
deployed to machine 133 until the running rc1 extraction/cache/analysis/report
validation is complete and this candidate has passed its own package gates.

## Candidate changes

- Adds configurable safe ZIP extraction concurrency (`auto`, 1, 2, 4 or a
  custom bounded worker count), deterministic serial fallback and a synthetic
  local benchmark. Source archives remain immutable and disk-capacity gates
  remain mandatory.
- Keeps the user-facing cache-prebuild module introduced by rc1 and adds the
  packaging checks needed to prove the updated operator controls are actually
  present in the frozen executable.
- Adds explicit one-sided manual cleaning controls: remove values below a
  draggable lower limit or above a draggable upper limit, with exact module and
  point identity, preview counts, optional time range and undo.
- Adds second-precision effective ranges for zero-offset correction. Pure legacy
  date-only end values still mean the whole day; explicit midnight remains an
  exact midnight boundary.
- Adds global/module/point time-series gap rendering inheritance. Point settings
  override module settings, which override compatible historical settings and
  then the global default. This changes line rendering only, not statistics.
- Adds a shared Python/MATLAB gap alias contract covering wind, earthquake,
  high/low-pass dynamic strain and both spectrum families.
- Adds line, branch, decision and condition coverage baselines without treating
  one arbitrary percentage as the release decision.
- Strengthens the frozen-package contract so an old eight-page GUI cannot be
  republished as this candidate. The manifest and release package must prove all
  four operator features and include a dedicated extraction-settings screenshot.
