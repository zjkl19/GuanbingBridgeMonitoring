# v1.8.2-dev

`v1.8.2-dev` is the development line after the stable `v1.8.1` release. It is
not a published production release and must not replace the immutable
`v1.8.1` package until its native Windows interaction and complete release
gates pass.

## Direct MATLAB FIG threshold tools

- The unified workbench restores the historical MATLAB workflow for choosing
  an arbitrary trusted `.fig` file and setting cleaning thresholds directly
  from the curve. This is a separate, operator-visible path alongside current
  task previews and the advanced workbench JSON import.
- Double-sided dragging keeps samples between the lower and upper lines. The
  two box tools remain independent: the lower-side box uses the highest finite
  sample in the rectangle as the lower threshold; the upper-side box uses the
  lowest finite sample as the upper threshold. Values equal to a threshold are
  retained.
- An external FIG is only a numerical reference for the currently selected
  point-specific configuration row. Its source curve identity is displayed
  and recorded but is never treated as proof that the FIG belongs to the
  current task. The existing time window is retained by default.
- The filtered post-cleaning editor exposes direct FIG interaction without
  silently reusing the pre-filter task-preview or JSON workflow.
- JPG and PNG files are not accepted by this entry because they do not retain
  the original curve coordinates required for reproducible threshold picking.
- A dedicated waiting dialog offers "Stop this FIG operation". It owns and can stop
  only the exact Runner created for that interaction, leaving formal analysis
  jobs and the source FIG untouched.

## Integrity and release gates

- The workbench records and verifies the selected FIG path, byte count,
  modification time and SHA-256 before and after interaction. A changed source
  file or inconsistent Runner result fails closed.
- Compiled-Runner smoke coverage exercises double-line, lower-box and upper-box
  requests against a controlled FIG fixture and verifies that the source FIG
  remains unchanged.
- The integrated release-health gate runs the complete Python suite, all
  configuration linters and every report-template precheck. MATLAB test
  discovery is partitioned without overlap into the non-GUI core suite and the
  verified-CSV cleanup contracts; native GUI cases run in their own isolated
  MATLAB process. Exact counts and timings are recorded by the health-check
  artifact generated from the release commit instead of being duplicated as
  release-note constants. The dedicated GUI process also passes
  the complete workbench smoke (`13` tabs, `16` tables and `6` path profiles).
  Focused FIG MATLAB `11/11`, operator-guide/package contracts, EXE build and
  compiled-Runner three-operation smoke also pass. A historical
  Hongtang Q2 FIG with `10,072` finite samples also passed read-only loading with
  unchanged SHA-256, byte count and timestamp.
- The native Windows manual interaction check passed on 2026-07-17. The
  operator selected the axes and curve in the controlled system-curve preview,
  dragged both threshold lines, used the lower and upper box tools, cancelled
  each edit, and returned focus to the parent workbench. The two drag values
  changed from `[-300, 300]` to about `[-87.51, 172.50]`; the controlled box
  fixture produced lower/upper candidates `-3` and `3`. Every dialog was
  cancelled, so the persisted configuration remained `[-300, 300]`. Direct
  arbitrary-FIG loading and the compiled MATLAB selector remain covered by the
  independent Runner integration gate.
- Cancelling the workbench's native MATLAB FIG wait window now writes a bound
  terminal `cancelled` status. A terminated child can no longer leave a stale
  `running` record that looks like an active task after the window is gone.
- Release-health checks no longer run GUI tests inside the long-lived full
  MATLAB test process. Non-GUI tests and GUI smoke use separate MATLAB
  processes and separate temporary `MATLAB_PREFDIR` directories. The GUI
  preflight ignores entries that Windows already marks as exited, blocks on a
  genuinely live interactive or background MATLAB session, and never restarts
  MathWorks services or terminates an unrelated MATLAB process. Timeout and
  failure cleanup is limited to the exact process tree started by the check.
- This isolation fixes the observed `MATLABWindow.exe`/`libcef.dll`
  `0x80000003` popup left by the former mixed full-suite GUI run. The isolated
  native GUI regression passed all 4 GUI tests and the complete workbench GUI
  smoke without leaving a live MATLAB or MATLABWindow process.

## Verified extracted-CSV cleanup

- The cleanup remains default-off and requires the exact
  `DELETE_VERIFIED_EXTRACTED_CSV` token. Original ZIP archives, WIM, Excel,
  unconfigured CSVs, MAT/meta cache pairs and durable receipts remain
  protected.
- A natural-day authorization now binds the exact CSV allowlist, source CRCs,
  MAT/meta cache pairs, ZIP inventory, effective configuration, date and lock
  owner. Each source/cache/ZIP payload is deeply verified once before the
  immutable authorization is published; the following rename/delete phase is
  a lightweight exact-allowlist transaction instead of repeating full ZIP and
  MAT reads for every file.
- Extraction, streaming cache work, fresh cleanup and interrupted cleanup
  resume share the same daily mutation lock. Interrupted or legacy receipts
  still fail closed into strict deep recovery, including the four supported
  filesystem recovery states.
