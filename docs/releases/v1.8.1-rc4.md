# v1.8.1-rc4

`v1.8.1-rc4` is the candidate that adds an opt-in, transaction-safe way to
reclaim disk space after pre-generating analysis caches for the daily ZIP
export layout used by Jiulongjiang and Shuixianhua. It remains disabled by
default. Stable `F:\Guanbing` and source ZIP trees are outside this candidate's
write boundary.

## Candidate changes

- Adds one-natural-day processing for `unzip + cache_prebuild + verified CSV
  cleanup`: extract, build/reuse the configured cache pairs, prove those MAT
  pairs are independently readable, prove the unchanged source ZIP can restore
  the exact CSV bytes, write a durable receipt, then stage and delete only the
  verified extracted CSVs.
- Requires the exact `DELETE_VERIFIED_EXTRACTED_CSV` confirmation token and a
  dedicated preprocessing task. The feature cannot be combined with CSV
  rename, header removal, resampling, analysis or report modules.
- Retains source ZIPs, WIM files, Excel workbooks, unconfigured CSVs and all
  cache/receipt evidence. A same-schema CSV that is not resolved from the
  active bridge point configuration is never eligible for deletion.
- Rechecks current source and staged-file CRCs immediately before rename and
  deletion. Same-size, same-timestamp replacement content therefore fails
  closed instead of passing a metadata-only comparison.
- Uses shared day-mutation, cache-build and partition-cleanup leases with a
  fixed lock order. Interrupted receipts are reconciled before re-extraction;
  committed days can be reused without restoring deleted CSVs.
- Adds a high-risk GUI control with an explicit explanation, default-off state,
  typed confirmation, saved-task round trip and release-smoke evidence. The
  option is available only for `jlj_daily_export` profiles.
- Keeps ordinary cache prebuild behaviour unchanged: when cleanup is not
  explicitly enabled, all source files remain in place.

## Validation boundary

- Release acceptance requires complete Python and MATLAB regression, strict
  GUI/CLI/build/package checks, compiled Runner tests for both the default-off
  and fail-closed policies, and a reviewed Windows screenshot of the control.
- Offscreen screenshots may be used while the development desktop is occupied,
  but they do not replace a later native focus/icon/DPI release check.
- Deployment is permitted only to the isolated
  `F:\Guanbing_v1.8.1-rc1` candidate tree after all gates pass. Existing rc3
  analysis may finish before the candidate is replaced; stable production is
  not upgraded by this release step.
