# 测试数据说明

此目录用于存放精简后的真实测试数据，便于单元测试/快速回归。请保持与真实数据一致的日期层级与子目录命名，但尽量控制文件体积。

## 建议目录结构
- 2025-01-01/ （按需 1~3 天即可）
  - 波形_重采样/        # 加速度（处理后）
  - 波形/              # 加速度原始波形（频谱/高频分析）
  - 特征值_重采样/      # 挠度
  - 特征值/            # 应变、裂缝、温度、湿度等共用
- 2025-01-02/
  - …（同上）

## 文件命名模式（需与 config/file_patterns 对齐）
- 挠度/应变/加速度等：`{point}_*.csv`
- 裂缝温度：`{point}-t_*.csv`

## 内容与格式要求
- 时间格式：`yyyy-MM-dd HH:mm:ss.SSS`，表头需包含 `header_marker`：`[绝对时间]`。
- 行数：50~200 行/文件；加速度原始波形可 200~1000 行（20–50 Hz，10–20 秒）。
- 数值：以正常范围为主，穿插少量超限与 0 值，用于验证阈值、`zero_to_nan`、`outlier` 清洗逻辑。

## 提示
- 目录已在 `.gitignore` 排除；仅 README.md/.gitkeep 会被提交。
- 如调整子目录名或命名模式，请同步更新 `config/*.json`。
