# 测试数据说明

此目录用于存放用于单元测试与回归的精简数据。
- 示例/手工对照数据可放在 `tests/data/` 下。
- 自动生成的单元测试最小数据放在 `tests/data/_unit/`。

## _unit 目录说明
`tests/data/_unit/` 由测试生成，包含最小可用 CSV：
- 覆盖 UTF-8 与 UTF-16LE(BOM) 读取
- 覆盖阈值过滤、时间窗过滤
- 覆盖 file_id / file_patterns 映射（wind/eq/cable_accel）
- 覆盖缺失文件、NaN 等边界情况

如需人工校对，可直接查看该目录下 CSV 内容。

## 建议目录结构（示例）
- 2025-01-01/
  - 波形_重采样/         # 加速度（处理后）
  - 波形/               # 加速度原始波形（频谱/高频分析）
  - 特征值_重采样/       # 挠度
  - 特征值/             # 应变、裂缝、温度、湿度等共用
- 2025-01-02/
  - …（同上）

## 文件命名模式（需与 config/file_patterns 对齐）
- 挠度/应变/加速度等：`{point}_*.csv`
- 裂缝温度：`{point}-t_*.csv`

## 内容与格式要求
- 时间格式：`yyyy-MM-dd HH:mm:ss.SSS`
- 表头需包含 `header_marker`，例如：`[绝对时间]`
- 行数建议：
  - 特征值：50~200 行/文件
  - 原始波形：200~1000 行（20~50 Hz，10~20 秒）
- 数值范围以正常区间为主，可少量插入异常值用于验证阈值/zero_to_nan/outlier 逻辑

## 提示
- `tests/data/_unit/` 允许提交到仓库，用于单元测试。
- 其他目录的数据如果较大，可继续保持在 `.gitignore` 内。
